How This System Works
01
Start here — every morning
Open this Master Hub first. Check the Org Pulse in the header. Go to Alerts tab if anything is red. Then open your role's Command Center for your personal action list.
→ CEO Command Center (your personal dashboard) → Alerts tab (what needs action today)
02
Every role has its own Command Center
Each of the 14 roles has a personal dashboard — KPIs, what's due today, what they're waiting on, their escalation path. Click any role card to open their Command Center.
→ All Roles tab to see every command center
03
Each role links to 3 more docs
From any Command Center you can reach: (1) the Cadence doc — daily/weekly tasks, (2) the Pillar Process Bible — how each process runs, (3) the RACI — who owns what org-wide.
→ All Documents tab for the full library
04
Pillar tabs show the full engine
Pillar 1 tab = Revenue Engine health. Pillar 2 = Delivery Engine. Pillar 3 = Operations. Each shows all roles in that pillar, their KPIs, and cross-pillar handoff status.
→ Pillar 1 · Revenue tab → Pillar 2 · Delivery tab → Pillar 3 · Ops tab
05
The document hierarchy
Master Hub → Role Command Center → Cadence Doc → Pillar Process Bible → RACI. You should never need to go deeper than 2 clicks from here to find any process, task, or decision rule.
→ All Documents tab for the full index
06
Updating the system
KPIs in this hub are updated weekly by pillar heads (VP Alliances, Delivery Head, Admin Head) and submitted to CEO. The data here reflects Friday reports. Stale data = the reporting process broke.
→ CEO Command Center for the full live view
